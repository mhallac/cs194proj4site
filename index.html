Overview

For this project, I use neural networks to automatically detect key points on faces.

Full facial keypoints detection

Now, given a face I try to detect 58 keypoints

The architecture of my example was as follows:

  • Convolution layer from 1 -> 16 with a kernel size 3x3, follwed by a relu and a max pool of size 2x2
  • Convolution layer from 16 -> 16 with a kernel size 3x3, followed by a relu and a max pool of size 2x2
  • Convolution layer from 16 -> 32 with a kernel size 3x3, followed by a relu and a max pool of size 2x2
  • Convolution layer from 32 -> 64 with a kernel size 3x3, followed by a relu and a max pool of size 2x2
  • Convolution layer from 64 -> 64 with a kernel size 3x3, followed by a relu and a max pool of size 2x2
  • A fully connected layer from 960 -> 960
  • Another fully connected layer from 960 to the output size of 58*2 = 136
My hyperparameters were to run it for 30 epochs with a learning rate of .001. In the first failure case above, the network detected the eye as the nose, and in the second failure case, I believe the algorithm saw the shadow under the mans chin and followed that to get an incorrect outline of the face.

Train with a larger dataset

For the larger dataset, I used the Resnet18 from pytorch with some modifications. The architecture consisted of the following:

  • Convolution from 1 -> 64 with a kernel size of 7x7 and a stride of 2, with a padding of 3
  • Batching followed by relu and maxpool
  • Layer 1: convolution, batching, relu, convolution, batching repeated twice
  • Layer 2: same as layer 1 with a downsample and additional convolution in the middle
  • Layers 3-4: Repeats of above
  • An average pool, followed by a fully connected layer
